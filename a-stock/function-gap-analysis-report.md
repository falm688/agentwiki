# A股量化交易系统 — 功能缺口分析报告

> **结论：当前系统已具备核心交易数据管道+可视化仪表盘，但存在5个P0级功能缺口直接影响交易决策，6个P1级缺口大幅影响体验，4个P2级锦上添花。核心痛点在于「市场状态分类器尚未完成→ETF动量轮动无法激活」「ML因子可见但不可用」「实盘与模拟盘脱节」。**

---

## 评估范围

| 维度 | 已审查内容 |
|:----|:----------|
| 前端页面 | 总览、策略列表/详情、短线打板、策略对比、自选股、股票查询、策略实验室（共9个Vue文件） |
| 后端API | overview/strategies/stock/backtest/lab/watchlist（6个FastAPI router） |
| 策略引擎 | V4超跌反转、V6+TR2、宽基动量、短线三模式、ML Alpha因子、气候分类器V3.1、ETF动量轮动 |
| 数据管道 | 资金流因子(7个)、融资融券因子(4个)、EOD数据同步、14:50实时执行cron |
| 知识库 | 22个文档（策略文档/回测验证/因子管线/模拟盘审计/架构文档） |

## P0 — 核心缺失（直接影响交易决策）

### P0-1. 市场状态分类器 → ETF动量轮动未激活

| 项目 | 内容 |
|:---|:-----|
| **缺失内容** | 气候分类器V3.1（在Lab中标记为30%进度）尚未完成，TREND模式（占24%时间）的ETF动量轮动已验证（+12.2%年化/-0.2%回撤）但处于休眠状态。当前V6+TR2仅在V4模式下运行，气候分类器打开后会使得TREND期空仓，全年错过约24%的ETF收益窗口。 |
| **为什么重要** | 天时V4完整架构中V4模式占57%、TREND模式占22%、CASH模式占22%。TREND模式缺失意味着近1/4的时间策略无法参与ETF动量行情，直接拉低全周期年化。且CASH保护模式也无法启用（需与TREND模式联动部署）。 |
| **建议实现路径** | ① 完成气候分类器V3.1的ret5反弹检测参数验证（已修复死区bug）→ ② 将ETF动量轮动脚本集成进主回测引擎（已验证+12.2%）→ ③ 启用分类器，在V4/TREND/CASH三模式间自动切换 → ④ 前端新增「市场状态指示器」显示当前模式及切换记录 |

### P0-2. ML Alpha因子不可用于决策

| 项目 | 内容 |
|:---|:-----|
| **缺失内容** | ML Alpha因子（XGBoost横截面评分，24个技术特征，163个模型，Rank IC=0.1121，IR=2.77）已完成85%、全量训练完成，但：① 前端无任何展示；② 未集成到短线候选股筛选流程；③ 无法回测ML因子叠加效果。用户知道它有，但用不上。 |
| **为什么重要** | 用户短线三模式（一进二/弱转强/N字）需要选股辅助信号。当前靠简单的换手率/竞价涨幅过滤，ML因子提供横截面排序能力，可显著提升短线候选股质量。 |
| **建议实现路径** | ① 后端新增 `/api/v1/factors/ml-alpha` 接口返回每日股票Alpha评分排名 → ② 前端新增「ML Alpha评分」页面：评分分布热力图+Top/Bottom榜+评分趋势 → ③ 集成到短线打板流程：候选股显示ML评分作为辅助列 → ④ 提供回测对比（纯规则 vs 规则+ML过滤） |

### P0-3. 实盘持仓与模拟盘脱节

| 项目 | 内容 |
|:---|:-----|
| **缺失内容** | 仪表盘仅跟踪模拟盘持仓（`sim_positions_v2`表），用户实际在同花顺/券商持有002328.SZ等实盘股票。自选股页写死了WATCHED_CODES=['002328.SZ']但无实盘价格同步、无合并资产视图。用户需要同时看模拟盘+实盘的完整资产状况。 |
| **为什么重要** | 用户目标年化20-25%含实盘部分。当前系统只反映模拟盘（10万资金）表现，无法判断整体资产配置是否达成目标。短线打板模式在同花顺独立操作，系统完全看不到。 |
| **建议实现路径** | ① 新增「实盘账户」配置：允许用户手动录入或通过飞书指令记录仓位 → ② 后端新增 `actual_positions` 表/路由，关联到同花顺模拟盘 → ③ 总览页合并显示「实盘+模拟盘」总资产、总盈亏 → ④ 短线打板页增加「今日操作备忘」模块记录同花顺操作 |

### P0-4. 盘中实时异动监控缺失

| 项目 | 内容 |
|:---|:-----|
| **缺失内容** | 系统仅在14:50（cron执行买入信号）和18:30（数据补录）运行。整个交易时段（9:30-15:00）无任何价格监控、量价异动提醒、板块轮动追踪。用户无法在盘中获知持仓股异动或市场突发变化。 |
| **为什么重要** | 用户持有002328.SZ且做短线打板，盘中异动（拉涨停/砸跌停/板块异动）是短线操作的关键决策依据。错过盘中信号意味着买入点严重滞后。 |
| **建议实现路径** | ① 利用腾讯实时行情API（`qt.gtimg.cn`，已有realtime.py模块）建立轮询服务（例：每5分钟拉取持仓+自选股实时价）→ ② 前端新增独立「实时监控」页或嵌入总览的实时价格栏 → ③ 设置价格阈值预警（涨超5%/跌超3%/成交量异常）→ ④ 飞书推送盘中异动通知 |

### P0-5. 股市舆情数据未集成

| 项目 | 内容 |
|:---|:-----|
| **缺失内容** | 用户关注淘股吧/雪球/东方财富的股市舆情来辅助判断，但系统中完全没有舆情数据集成。前端无任何新闻/论坛热帖/概念热度面板。虽然有Tushare新闻API可用，但未实现。 |
| **为什么重要** | 短线打板决策高度依赖市场情绪和热点题材。用户需要通过舆情判断板块热度（哪个概念在发酵）、龙头股（封板质量）、市场情绪（涨停家数/连板高度）。当前全靠手工在外部网站查看。 |
| **建议实现路径** | ① 利用Tushare财联社板块异动（`cls_market_shock`）+ 东方财富热榜（`dc_hot`）+ 同花顺热榜（`ths_hot`）API收集数据 → ② 新增「市场情绪」页面：涨停家数/炸板率/连板天梯/概念热度排行 → ③ 自选股/持仓股关联新闻摘要 → ④ 每日早报自动汇总前日舆情热点 |

## P1 — 重要增强（大幅提升体验）

### P1-1. 每日早报推送未上线

| 项目 | 内容 |
|:---|:-----|
| **缺失内容** | 用户提到的「每日早报（即将上线）」尚未实现。复盘推送已有（v6tr2_weekly_review），但开盘前需要的市场状态摘要、昨日信号回顾、今日候选股清单尚未自动化。 |
| **为什么重要** | 用户操作手册要求每日9:25-9:30查看监控报告。没有早报意味着用户需要手动打开多个页面合并信息，容易遗漏信号。 |
| **建议实现路径** | ① 后端新增早报组装逻辑（大盘状态+昨日交易回顾+今日候选股+持仓异动）→ ② cron定时8:30生成 → ③ 飞书机器人推送 → ④ 前端「消息中心」页存档历史早报 |

### P1-2. 策略参数无可视化配置面板

| 项目 | 内容 |
|:---|:-----|
| **缺失内容** | 所有策略参数（ED=8%、TP=16%、SL=12%、HD=25天、MAX=3只、TR_min=2.0%、vol20_low=20、stab_filter=-0.05等）硬编码在Python脚本中。修改参数需SSH+编辑代码+重启cron。无UI化配置。 |
| **为什么重要** | 用户迭代策略需要反复调整参数。当前方式不友好且容易出错，升级策略（如V6→V7切换）需要改代码，用户无法自助调试。 |
| **建议实现路径** | ① 后端新增策略配置表 + CRUD API → ② 策略详情页增加「参数配置」Tab（滑块/下拉框形式展示所有参数）→ ③ 参数修改后自动生成回测预览 → ④ 参数变更历史记录 |

### P1-3. 回测参数不可定制

| 项目 | 内容 |
|:---|:-----|
| **缺失内容** | 后端回测引擎（`core/backtest.py`）功能强大但前端策略实验室仅显示静态回测结果。用户无法选择回测时段、调整参数、对比不同配置的绩效。 |
| **为什么重要** | 用户无法验证\"如果我改了参数会怎么样\"——这是策略迭代的核心需求。当前只能通过命令行跑回测脚本再看结果，反馈周期长。 |
| **建议实现路径** | ① 后端新增 `/api/v1/backtest/run` 接口接受参数并调用回测引擎 → ② 策略实验室增加「自定义回测」Tab：参数表单+时间段选择+立即运行 → ③ 回测结果可视化（净值/回撤/逐年收益/归因分析） |

### P1-4. 资金流向因子数据无呈现

| 项目 | 内容 |
|:---|:-----|
| **缺失内容** | 资金流因子管线已完成（7个因子、7,137,873行数据），融资融券因子管线也在建设中（4个因子）。但前端除个股详情页的简单资金流向图外，没有任何因子数据展示面板。用户无法按因子筛选/排序股票。 |
| **为什么重要** | 这些因子已证明对V4策略无叠加价值（6M行验证），但作为辅助分析工具仍有价值——例如查看\"融资余额变化率\"判断杠杆情绪，\"大单净额\"识别主力动向。当前数据在沉睡。 |
| **建议实现路径** | ① 后端新增 `/api/v1/factors` 路由返回每日因子截面数据 → ② 前端新增「因子浏览器」页：选择因子+时间+排序，展示全市场Top/Bottom → ③ 个股详情页增加因子Tab → ④ 因子与策略信号的关联分析 |

### P1-5. 预警/通知配置无UI

| 项目 | 内容 |
|:---|:-----|
| **缺失内容** | 飞书推送由cron固定触发，用户无法在界面中配置：关注哪些股票/触发什么条件/通知频率/启用或禁用特定推送。所有配置需改后端代码。 |
| **为什么重要** | 用户监控的是多维度信息（持仓股价格、模拟盘净值变化、短线候选股触发），需要按需配置推送阈值和内容，避免信息过载。 |
| **建议实现路径** | ① 新增 `alert_config` 表存储用户预警设置 → ② 前端新增「预警设置」页：股票价格预警/策略信号预警/净值变化预警 → ③ 后端检查引擎在cron执行时评估条件 → ④ 飞书定向推送 |

### P1-6. 宽基动量ETF轮动无专项视图

| 项目 | 内容 |
|:---|:-----|
| **缺失内容** | 宽基动量策略是三并行策略之一，但前端没有任何视图展示：当前选择的ETF、动量评分排名、轮动历史、与基准对比。策略详情页只显示通用净值/持仓。 |
| **为什么重要** | 用户需要理解\"为什么选了这只ETF\"——动量评分的变化趋势、换仓点的决策逻辑、与其他ETF的对比。缺乏透明度的策略用户不会信任。 |
| **建议实现路径** | ① 后端新增动量评分历史API → ② 策略详情页增加ETF动量子面板：4指数动量计分卡（雷达图/条形图）+ 持仓历史时间线 + 每次换仓的评分对比 → ③ 结合天气分类器状态显示当前模式 |

## P2 — 锦上添花

### P2-1. 短线打板每日候选股看板

| 项目 | 内容 |
|:---|:-----|
| **缺失内容** | 后端短线的 `get_candidates()` 从 `/tmp/da_ban_candidates.json` 读取每日候选股，但前端没有对应页面。只有历史回测结果展示。 |
| **建议实现路径** | ShortTerm.vue增加「今日候选」Tab：竞价涨幅/换手率/ML评分/行业热度多维展示 → 点击跳转个股详情 → 带手动添加自选功能 |

### P2-2. 收益归因分析

| 项目 | 内容 |
|:---|:-----|
| **缺失内容** | 无法将收益拆解到：各策略贡献、各持仓贡献、不同市场状态下表现、不同行业/概念贡献。逐年收益分解在wiki文档里有但UI中无。 |
| **建议实现路径** | 后端新增归因分析接口 → 策略对比页增加归因Tab可选择归因维度（时间/策略/行业/因子）→ Brinson归因分解（选股效应+配置效应） |

### P2-3. 多策略资金分配模拟器

| 项目 | 内容 |
|:---|:-----|
| **缺失内容** | Lab中标记15%进度的项目。用户有三并行策略+短线模式，但没有工具模拟\"70%给深跌企稳+30%给换手率\"之类的资金分配效果。 |
| **建议实现路径** | 策略实验室增加「资金分配模拟」模块：拖拽分配各策略权重 → 合并净值曲线预览 → 风险指标对比（组合夏普/最大回撤） |

### P2-4. 全量操作日志与审计追踪

| 项目 | 内容 |
|:---|:-----|
| **缺失内容** | `sim_trades_v2` 表记录交易但埋得深。前端无法跨策略搜索/过滤历史交易、查看交易分布（按月份/按股票/按盈亏区间）。无审计视图。 |
| **建议实现路径** | 新增「交易日志」页面：统一展示所有策略交易记录 → 高级过滤（策略/方向/盈亏/日期范围）→ 交易分布统计图表 → 盈亏贡献排名（哪只股票赚最多/亏最多） |

## 汇总

### 优先级分布

| 优先级 | 数量 | 关键特征 |
|:-----|:---:|:--------|
| **P0** | 5 | 直接影响交易决策能力，缺一个就是功能性残疾 |
| **P1** | 6 | 大幅提升使用体验和策略迭代效率 |
| **P2** | 4 | 锦上添花，增强分析深度 |
| **合计** | **15** | |

### 时间线建议

```
立即（1-2天）：P1-1 早报上线 + P0-5 舆情数据基础接入
短期（1周）：P0-1 分类器完成 → ETF轮动激活 + P0-2 ML因子可视化
中期（2周）：P0-3 实盘同步 + P0-4 实时监控 + P1-2/3 参数面板
长期（1月+）：P1-4/5/6 + P2级所有功能
```

### 总体评价

当前系统的后端数据管道和策略引擎（V4/V6/ML/因子管线）质量较高，但**数据到决策的最后一公里**大量缺失。用户有很好的策略但缺乏工具来执行、监控和优化。5个P0缺口补全后，系统将从「可视化监控面板」进化为一套真正闭环的「量化交易决策平台」。
