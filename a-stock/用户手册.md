## 1. 系统架构

```
                    ┌──────────────────────┐
                    │     MySQL qteasy_db   │
                    │  stock_daily (758万行)│
                    │  daily_basic          │
                    │  fund_daily (ETF)     │
                    │  trade_calendar       │
                    └──────────┬───────────┘
                               │
              ┌────────────────┼────────────────┐
              ▼                ▼                 ▼
      ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
      │ BacktestEngine│ │  SimV2      │ │  实时行情    │
      │ (回测历史)    │ │ (模拟盘)    │ │ (腾讯API)    │
      └──────────────┘ └──────────────┘ └──────────────┘
              │                │                 │
              ▼                ▼                 ▼
      ┌─────────────────────────────────────────────┐
      │        策略类 (BaseStrategy子类)             │
      │  tianshi_tr / tianshi_stab / etf_momentum  │
      └─────────────────────────────────────────────┘
```

**两层架构：**
- **回测层** — `BacktestEngine` + `DataLoader`，读MySQL历史数据，按日模拟交易
- **模拟盘层** — `SimV2` + `fast_loader` + 腾讯实时API，每日14:50执行

两个引擎共享同一套策略类（`BaseStrategy.generate_signals()`）。

## 3. 策略参数一览

### 3.1 天时·换手率

| 参数 | 默认值 | 说明 |
|:----|:-----:|:-----|
| `ed` | 0.15 | 60日跌幅门槛（-15%）|
| `tr_min` | 2.0 | 换手率下限（>2%）|
| `tp` | 0.16 | 止盈（+16%）|
| `sl` | 0.12 | 止损（-12%）|
| `hd` | 25 | 最长持有天数 |
| `max_pos` | 3 | 最大并发持仓 |
| `top_n` | 9 | 每日候选股上限 |

**逻辑：** 从全市场选出 `r60 <= -15%` 且 `turnover_rate > 2%` 的股票，按超跌程度+换手率排序，取前top_n，实际买入最多max_pos只。

**选股数据依赖：** `turnover_rate`（来自 `daily_basic` 表）

### 3.2 天时·深跌企稳

| 参数 | 默认值 | 说明 |
|:----|:-----:|:-----|
| `ed` | 0.15 | 60日跌幅门槛（-15%）|
| `ret5_threshold` | -0.05 | 近5日涨幅门槛（>-5%，即企稳）|
| `tp` | 0.16 | 止盈（+16%）|
| `sl` | 0.12 | 止损（-12%）|
| `hd` | 25 | 最长持有天数 |
| `max_pos` | 3 | 最大并发持仓 |
| `cooldown_days` | 30 | 同一只股票卖出后冷却期 |
| `top_n` | 9 | 每日候选股上限 |

**逻辑：** 从全市场选出 `r60 <= -15%` 且 `ret5 > -5%`（近5日企稳）的股票，按超跌程度排序。卖出后30天内不再买入同一只。

### 3.3 宽基动量（ETF轮动）

见 `strategies/etf_momentum/config.json`。

**逻辑：** 4只宽基ETF（沪深300/中证500/创业板/上证50），计算20日动量，买入动量最强的ETF。每月换仓阈值5%。

**注：** ETF数据来自 `fund_daily` 表，不是 `stock_daily`。

### 3.4 短线打板

见 `strategies/short_term/config.json`。

**模式：** 一进二 / 弱转强。仅限主板10cm（排除ST/科创板/创业板）。

## 4. 运行回测（标准流程）

### 4.1 使用 BacktestEngine（推荐）

```python
from core.backtest_engine import BacktestEngine
from core.data_loader import DataLoader
from strategies.tianshi_tr.strategy import TianshiTRStrategy

loader = DataLoader()
engine = BacktestEngine(data_loader=loader, initial_cash=100000)
strategy = TianshiTRStrategy()
result = engine.run(strategy, '2021-01-01', '2025-12-31')

print(f'总收益: {result.total_return:.2f}%')
print(f'年化: {result.annual_return:.2f}%')
print(f'最大回撤: {result.max_drawdown:.2f}%')
print(f'胜率: {result.win_rate:.2f}%')
print(f'交易次数: {result.total_trades}')
print(f'逐年收益: {result.yearly_returns}')
```

### 4.2 已写好的回测脚本

```bash
cd ~/profiles/alex/home/trading
source ../venv/bin/activate

# 三策略对比（2021-2026）
python3 compare_annual.py

# 定制回测
python3 -c "
from core.backtest_engine import BacktestEngine
from core.data_loader import DataLoader
from strategies.tianshi_stab.strategy import TianshiStabStrategy
engine = BacktestEngine()
result = engine.run(TianshiStabStrategy(), '2023-01-01', '2024-12-31')
print(result)
"
```

### 4.3 注意事项

- **天时·换手率** 需要 `turnover_rate` → `DataLoader` 已配置加载
- **宽基动量** 需要 `fund_daily` 数据 → 确保MySQL有该表
- 回测使用前复权价格（`adj`列）
- 手续费：买入万3、卖出万13（含印花税）
- 滑点：±1%

## 5. Cron任务与自动化

| 时间 | 任务 | 脚本 | 说明 |
|:----|:----|:-----|:-----|
| **09:00** | 📰 投资早报 | LLM Cron → 飞书推送 | 大盘/行业/资金/外围/持仓 |
| **09:25** | 🔍 竞价过滤推送 | 读候选JSON → 实时竞价过滤 | 短线打板候选 |
| **14:35** | 🔥 缓存预热 | `warm_cache_1435.sh` | 防止14:50缓存过期超时 |
| **14:50** | 📈 天时·换手率 | `execute_tr_1450.sh` | 实时价买入/卖出 |
| **14:52** | 📈 天时·深跌企稳 | `execute_stab_1450.sh` | 同上 |
| **14:54** | 📈 宽基动量 | `execute_mom_1450.sh` | ETF轮动 |
| **15:30** | 📋 尾盘复盘 | LLM Cron → 飞书推送 | 当日交易/健康检查 |
| **18:30** | 🔄 数据补录 | `sync_data_1830.sh` | Tushare拉T日数据，重试到23:00 |

**cron包装脚本位置：** `~/.hermes/profiles/alex/scripts/`

**注意：** `~/.hermes` 是软链 → `profiles/alex/home/.hermes`。

## 6. 数据源说明

### 6.1 MySQL `qteasy_db`

| 表名 | 数据量 | 日期范围 | 说明 |
|:----|:-----:|:---------|:-----|
| `stock_daily` | 758万行 | 2010→至今 | A股日线（open/high/low/close/vol/amount/pct_chg）|
| `daily_basic` | 90万行 | 2021→2024完整 + 2026-05 | **换手率/PE/PB/市值** ⚠️缺2025全年 |
| `fund_daily` | 有 | 视同步情况 | ETF日线数据 |
| `trade_calendar` | 有 | 全量 | 交易日历（SSE）|

### 6.2 日期格式陷阱

```
stock_daily.trade_date     →  '2023-06-01'  (YYYY-MM-DD)
daily_basic.trade_date     →  '20230601'    (YYYYMMDD)  ← 注意无横线!
trade_calendar.cal_date    →  '2023-06-01'
```

两个表JOIN时必须统一格式。

### 6.3 数据缺失

- `daily_basic` **缺2025年全年数据**（有2021-2024完整，和2026-05）
- 因此天时·换手率的"近1年"回测无法直接跑2025-2026
- 需要跑最早的有数据的完整年度：2023-06至2024-05，或2024年

## 7. 常见坑与解决办法

### 7.1 回测跑出0笔交易

**根因：** `DataLoader.load_daily()` 默认不加载 `turnover_rate`，天时·换手率策略依赖该字段。

**修复：** 调用 `DataLoader.load_daily()` 时 `include_factors=True`（默认已开启）。

### 7.2 日期格式不统一

**现象：** `daily_basic` 的日期格式是 `20230601` 而非 `2023-06-01`，直接JOIN `stock_daily` 会全空。

**修复：** 查询 `daily_basic` 后用 `pd.to_datetime(..., format='mixed')` 统一格式再JOIN。

### 7.3 ETF数据不存在

**现象：** 宽基动量回测报错或查询不到。
**根因：** ETF代码（如159915.SZ）不在 `stock_daily` 表。
**修复：** 使用 `fund_daily` 表。

### 7.4 Cron脚本软链陷阱

**现象：** 脚本写入 `profiles/alex/scripts/` 后Cron找不到。
**根因：** `~/.hermes` 是软链指向 `profiles/alex/home/.hermes`。
**修复：** 写入后 `cp` 到软链指向的路径，或直接用已存在的包装脚本。

### 7.5 fast_loader缓存过期超时

**现象：** 14:50三策略全部超时，收不到推送。
**根因：** `.adj_cache.pkl` 6h有效期，14:50执行时需重建缓存（耗时~150s）但Cron超时120s。
**修复：** 14:35预热缓存 `warm_cache_1435.sh`。如果缓存已过期，手动删 `.adj_cache.pkl` 再跑 `load_fast(refresh_cache=True)`。

### 7.6 涨停买不进/跌停卖不出

`trading_rules.py` 中的 `is_buy_possible()` 和 `is_sell_possible()` 会在价格接近涨停/跌停时阻止交易。回测和模拟盘都生效。
