> **一句话结论：市值20-200亿是唯一有效改进，年化+17.8%/回撤-20.8%，多因子叠加全部更差。**

## V7全因子扫描结果

| 因子 | 年化 | 回撤 | 结论 |
|:----|:---:|:----:|:----|
| 🏆 **MV 20-200亿** | **+17.8%** | **-20.8%** | ✅ **最优** |
| PE 10-50 | +17.0% | -23.1% | 接近 |
| V6基线（无因子） | +16.1% | -29.4% | 对照 |
| PB 0-5 | +15.3% | -24.2% | 一般 |
| MV+PB双因子 | ~+15% | -22~-25% | 不如纯MV |
| 所有三因子组合 | <+11% | — | ❌ 全部失败 |

**关键发现**：
- **只加一个因子就够了**：市值20-200亿过滤
- 叠加PE或PB → 候选池太小 → 收益下降
- 三因子组合全部跑输基线
- 所有结果使用**动态因子过滤**（FactorStore），无look-ahead bias

## 基础设施搭建（2026-06-09）

新建 `strategies/core/` 模块化架构：

| 模块 | 功能 |
|:----|:-----|
| `core/data.py` | 统一数据加载器，前复权缓存，共享数据 |
| `core/backtest.py` | 参数化回测引擎，dict入参/dict出参 |
| `core/factors.py` | FactorStore因子查询 + 插值 |
| `core/audit.py` | 稽核数据包编译 + delegate_task审计 |

**因子数据补全**：用 tushare Python SDK 批量拉取缺失字段（turnover_rate, volume_ratio, dv_ratio, ps, pe_ttm）回填 MySQL daily_basic表。新增列：dv_ratio, ps, pe_ttm, total_share, free_share。

**审计流程**：回测结果 → core/audit.py编译审计包 → delegate_task给auditor profile → 独立审核 → 返回PASS/WARN/FAIL结论。
