# 恒生公司 TA 与客服系统接口规范 V4.10

**一句话结论：登记过户系统向客服系统（CALL CENTER、网站查询）提供数据的接口规范，以文本文件定长记录方式传输。**

## 版本信息

- **版本**: V4.10
- **来源文件**: 恒生公司TA与客服系统接口规范V4.10.docx

## 数据处理规则

1. **文件格式**: 文本文件定长记录
2. **兼容性**: 新增字段增加在尾部，客服系统不校验记录长度
3. **文件结构**:
   - 第一行: 数据记录数
   - 最后一行: `END`
   - 中间: 每行一条完整记录
4. **换行符**: 回车(0DH) + 换行(0AH)
5. **数据项类型**:
   - `C`: 字符型
   - `N`: 数字
   - `A`: 字符型，填数字
6. **负数表示**: 第一位用 `-` 表示，占用一个整数位
7. **对齐规则**:
   - 数字: 左补零右对齐
   - 字符: 右补空格左对齐
8. **小数处理**: 传输时不传小数点，默认2位小数
9. **主索引**: 带下划线的字段

## 数据内容

包含六个方面数据：

| 数据类型 | 说明 |
|:---|:---|
| 客户账户信息 | 客户基本信息 + 基金账号登记信息 |
| 账户类确认数据 | 账户相关确认信息 |
| 客户基金份额余额 | 静态份额余额 + 份额明细 |
| 客户交易流水 | 申请流水 + 确认流水 |
| 基金分红信息 | 分红相关数据 |
| 基金行情信息 | 基金净值等行情数据 |

## 更新规则

- **频率**: 每日提供一次
- **范围**: 截至当日做完登记过户操作后的数据
- **模式**: 每次只提供变化（新增/更新）的数据；也可不定期提供全部数据（文件名区分）

## 客户账户信息数据

### 数据生成规则

- 客户基本信息和基金账号登记信息同步更新
- 客户资料和基金账号资料合在一条记录
- 支持新增客户记录、新增基金账号记录、修改已有记录（含销户）
- 基金账号登记信息变化时，该基金账号下所有登记信息全部更新

### 文件命名

| 数据类型 | 变更数据 | 全部数据 |
|:---|:---|:---|
| 客户基本信息 | `Acco_yyyymmdd.txt` | `Acco_yyyymmdd_all.txt` |
| 基金账号登记信息 | `Acconet_yyyymmdd.txt` | `Acconet_yyyymmdd_all.txt` |

## 客户基金份额余额数据

### 数据生成规则

- 基金静态份额余额和份额明细同步更新
- 静态份额: 只生成当天发生变化的记录（含修改、新增）
- 份额余额变为0也会生成对应数据

## 关键要点

- 客服系统需兼容字段扩展（不校验记录长度）
- 区分变更数据和全量数据（通过文件名后缀 `_all`）
- 负数第一位用 `-` 表示
