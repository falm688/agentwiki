# 回测引擎 V2 调试报告

## 问题

引擎（BacktestEngine）与独立脚本对同一策略跑出不同结果。

| 指标 | 独立脚本 | 引擎(修复前) | 修复后引擎 |
|:---|:---:|:---:|:---:|
| 4年收益 | +106.04% | +102.45% | +75.16% |
| 交易笔数 | 175 | 1,363 | 332 |
| 2022年收益 | -19.4% | +11.1% | ~0% |
| 最大回撤 | - | 34.5% | 20.7% |

## 根因分析

通过逐日逐笔对比脚本（`scripts/bt_compare.py`），定位到3个具体问题：

### 1. 首日跳过缺失 ⚠️

独立脚本有 `if i > 0` 跳过第一个交易日。引擎没有这个检查。

- 影响：引擎在第一个交易日买入股票，独立脚本跳过
- 结果：初始持仓不同 → 后续交易序列完全不同
- 修复：添加 `day_index == 0: continue` 跳过首日

### 2. MAX_POS 硬限制缺失 ⚠️

引擎的 `_execute_buys()` 没有检查 `len(positions) >= max_pos`。

- 影响：虽然策略 head(3) 限制了信号数量，但缺少防御性检查
- 修复：在买入循环前添加 `if len(positions) >= max_pos: break`

### 3. 涨停/跌停检查（NOT a bug）✅

引擎使用 `is_buy_possible` / `is_sell_possible` 检查涨停/跌停。独立脚本没有。

- 这不是bug，是引擎更接近实盘的表现
- 验证：1个月测试结果完全一致（期间无涨停/跌停事件）
- 6个月测试出现分歧：2月7日某股跌停，引擎延迟卖出，导致后续全部交易路径不同

## 验证方法

使用 `scripts/bt_compare.py` 逐日逐笔对比：
1. 1个月测试：7笔交易完全一致（参数对齐 + 无涨停跌停事件）
2. 6个月测试：前5笔完全一致，第6笔因跌停事件分叉
3. 验证了引擎的涨停/跌停检查正常工作

## 最终结果

```
天时·深跌企稳 | 2021-2024 | 4年
收益: +75.16% | 年化: 15.1% | 回撤: 20.7%
交易: 332笔 | 手续费: ¥4,817

费用: 万2.5佣金最低5元 + 万5印花税(卖出) + 过户费万0.1(沪)
滑点: 0.2%
```

## 结论

引擎核心逻辑正确，差异完全由实盘规则的差异解释。独立脚本（无涨停跌停检查）给出的是乐观估计（+106%），引擎（含实盘规则）给出的是实际预期（+75%）。
