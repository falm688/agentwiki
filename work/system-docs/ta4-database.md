# TA4.0 数据库设计说明

**一句话结论：恒生开放式基金登记过户系统V4.0的数据库设计，采用Oracle数据库，定义了统一的数据类型命名规范和表空间结构。**

## 版本信息

- **系统**: 恒生开放式基金登记过户系统 V4.0
- **来源文件**: TA4.0-数据库设计说明.xls
- **部门**: 基财三部

## 数据库对象

| 数据库/表空间 | 设备名 | 文件名 | 大小 | 说明 |
|:---|:---|:---|:---|:---|
| run2k | run2k_dat | d:\sqldata\run2kdat.mdf | 10MB | 系统当前库，存放当前表数据 |

## 设计要点

### 数据类型定义

仅使用3种数据类型：
- `char`
- `varchar2`
- `number`

### 数据类型约定

| 类型 | 约定 | 前缀 |
|:---|:---|:---|
| char | 写数据时必须写满长度 | `d_`（日期型）、`c_`（普通char） |
| varchar2 | - | `c_` |
| number(M,N) | 根据实际所需定义长度 | `f_` |
| number(M) | - | `l_` |

### 日期类型

- 统一使用 `char` 型
- 以 `d_` 形式表示

## 表分类

数据库设计说明包含以下表类别：

| 序号 | 类别 | 说明 |
|:---|:---|:---|
| 01 | 账户交易表 | 账户和交易相关核心表 |
| 02 | 交易类表 | 各类交易业务表 |
| 03 | 份额相关表 | 份额余额、明细等 |
| 04 | 分红收益表 | 分红、收益相关 |
| 05 | 信息维护表 | 客户信息、参数维护 |
| 06 | 统计表 | 各类统计报表 |
| 07 | 查询/报表/接口配置 | 配置类表 |
| 08 | 其他表 | 辅助表 |
| 09 | 债券股权登记 | 债券、股权相关 |
| 10 | 余额理财 | 余额理财业务 |
| 11 | 基金质押 | 质押业务 |

## 关键要点

- 统一使用 `char` 代替 `date` 类型
- 字段命名有明确前缀规范（`c_`, `d_`, `f_`, `l_`）
- Oracle 数据库，表空间名为 `run2k`
