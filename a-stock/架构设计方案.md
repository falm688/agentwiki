# 量化交易系统架构 — 修订版

> **一句话**：插件式策略 + 组合管理器 + 执行管理器 + 严格分离回测/模拟盘数据路径。打板和波段统一接入，多策略不冲突。

## 架构总览

```
                    ┌──────────────────────────────┐
                    │          Portforlio Manager    │ ← 全局组合管理
                    │  全局持仓 / 资金分配 / 冲突仲裁 │
                    └──────────┬───────────────────┘
                               │
         ┌─────────────────────┼─────────────────────┐
         ▼                     ▼                     ▼
   ┌───────────┐       ┌───────────┐       ┌───────────┐
   │ 波段策略A  │       │ 波段策略B  │       │ 打板策略C  │
   │ generate  │       │ generate  │       │ generate  │
   │ _signals  │       │ _signals  │       │ _signals  │
   │ (14:50)   │       │ (14:50)   │       │ (09:25+   │
   └─────┬─────┘       └─────┬─────┘       │  14:50)   │
         │                   │             └─────┬─────┘
         └───────────────────┼───────────────────┘
                             ▼
                    ┌────────────────┐
                    │ Execution Manager │ ← 信号→订单
                    │ 订单类型/价格条件  │
                    │ 虚拟成交/状态跟踪  │
                    └────────┬───────┘
                             ▼
                    ┌────────────────┐
                    │     Data       │
                    │    Layer       │ ← DataLoader封装
                    │  MySQL/Tushare │   策略不直接访问DB
                    └────────────────┘
```

## 一、六大体系（修订版）

### 1️⃣ 策略接口体系 — 支持多时间维度

```python
class BaseStrategy:
    """所有策略基类 — 支持日频和盘中两种信号模式"""
    
    # === 基本信息 ===
    strategy_id: str       # "tianshi_tr"
    name: str              # "天时·换手率"
    strategy_type: str     # "波段" | "打板" | "动量"
    version: str           # "v1.0"
    
    # === 核心方法 ===
    def calculate_indicators(self, date, data) -> pd.DataFrame:
        """计算策略所需的指标"""
        pass
    
    def generate_signals(self, date, data) -> List[Signal]:
        """每日一次信号（波段用）"""
        return []
    
    def generate_timeframe_signals(self, date, data, timeframe: str) -> List[Signal]:
        """按时间窗口信号（打板用）
        timeframe: 'open_call_auction'(09:25) | 'close'(14:50)
        """
        return []
    
    def get_config(self) -> dict:
        """返回当前参数"""
        return {}
```

**Signal** 类扩展：

```python
@dataclass
class Signal:
    code: str              # 股票代码
    action: str            # "buy" | "sell"
    price: float           # 目标价格
    price_type: str        # "limit"(限价) | "market"(市价)
    position_ratio: float  # 仓位比例 (0.1 = 10%)
    reason: str            # 信号理由
    strategy_id: str       # 来源策略
```

### 2️⃣ 回测体系 — 与模拟盘数据路径严格分离

```
回测引擎 ← 历史数据快照（一次性加载，时间截断）
模拟盘引擎 ← 实时/日终数据（每日增量）

同份策略代码，不同数据源
```

回测结果入库表：

| 表名 | 内容 |
|:-----|:------|
| `bt_summary` | 策略ID、总收益、年化、最大回撤、胜率、盈亏比、夏普、费比、交易次数 |
| `bt_trades` | 逐笔交易：日期、代码、方向、价格、收益、持有天数 |
| `bt_nav` | 日频净值曲线 |
| `bt_config` | 回测参数快照 |

支持任意两策略横向对比：

```sql
SELECT * FROM bt_summary WHERE strategy_id IN ('tianshi_tr', 'daban_yijin')
ORDER BY annual_return DESC;
```

### 3️⃣ 模拟盘体系 — 带组合管理

```
三层结构:

策略层 ← 独立运行，产生 Signal
     │
     ▼
Portfolio Manager ← 全局组合管理（新增✅）
     │  ├─ 全局持仓表（所有策略的总和）
     │  ├─ 资金分配（总本金 × 策略权重）
     │  ├─ 冲突仲裁（同股多信号 → 优先级/合并/拒绝）
     │  └─ 对冲禁止（同一只股票不能A买B卖）
     │
     ▼
Execution Manager ← 执行管理（新增✅）
     │  ├─ Signal → 订单转换
     │  ├─ 虚拟成交逻辑（打板需模拟排队）
     │  ├─ 订单状态跟踪
     │  └─ 写入状态文件
     │
     ▼
状态文件: sim_state_{strategy_id}.json
组合视图: sim_portfolio.json（所有策略汇总）
```

**Portfolio Manager 核心规则**：

| 场景 | 处理 |
|:----|:-----|
| 策略A和策略B同时买入同一只股 | 合并仓位，取先到者 |
| 策略A持有，策略B想卖出 | 不允许（对冲） |
| 所有策略总持仓超过组合上限 | 按策略权重等比压缩 |
| 总资金不够 | 按策略优先级分配 |

### 4️⃣ 因子体系 — 轻量级（延后实现）

先不建专门的因子系统。因子直接在策略的 `calculate_indicators` 中计算，存入MySQL `factor_values` 表。

待策略数量>5后再考虑正式的因子注册表。

### 5️⃣ 策略管理体系 — 简化为4态

```
研发(dev) → 回测(bt) → 模拟(sim) → 实盘(live)
                                    ↓
                                 废弃(archived)
```

策略注册表（MySQL）：

| 字段 | 说明 |
|:-----|:------|
| strategy_id | 唯一ID |
| name | 中文名 |
| type | 波段/打板/动量 |
| status | dev/bt/sim/live/archived |
| version | 版本号 |
| config | JSON参数 |
| created_at | 创建时间 |
| last_bt_summary | 最近回测绩效摘要 |

## 三、关键风险应对

| 风险 | 应对方案 |
|:-----|:---------|
| 回测-模拟盘偏差 | 数据路径严格分离：回测用历史快照，模拟盘用当日数据 |
| 打板未来函数 | 回测时按时间窗口截断数据，不回测当日未完成事件 |
| 多策略冲突 | Portfolio Manager 全局仲裁，单向仓位不变 |
| 打板时效性 | Cron定时 + 开机自检（先验证数据到位再执行） |
| Tushare限流 | 数据缓存 + 降级到腾讯API |
